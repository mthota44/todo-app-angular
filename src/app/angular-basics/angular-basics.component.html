The "Front Door" key.

Think of your app as a house. main.ts is the main entrance whereby Angular starts "bootstrapping" (loading) your application. It tells Angular which Component to show first (usually AppComponent).

üìÑ index.html

The Canvas.

Angular is a "Single Page Application" (SPA). This is that single page. It has a tag <app-root> where Angular injects your entire app. You rarely touch this file directly.

üìÑ app.config.ts

The Rule Book.

This is where we set up global rules, like "enable routing" or "setup HTTP client". Before standalone components, this was done in app.module.ts.

üìÑ angular.json

The Architect's Blueprint.

It tells the CLI how to build your app. Where are the assets (images)? What styles to include globally? Which index.html to use? It manages the workspace configuration.

üì¶ package.json & package-lock.json

The Shopping List & Receipt.

  • package.json: Lists what libraries your app needs (like "Angular", "Bootstrap"). It's like a shopping list.
  • package-lock.json: The detailed receipt. It locks the exact version of every library installed so everyone on the team has the exact same setup.

The Translator's Guide.

These files tell the TypeScript compiler how to behave. For example, "should we allow strict typing?" or "target modern JavaScript?".

  • app: For your actual app code.
  • spec: For testing code.
  • json (root): Base settings.

Why TypeScript? ü§î

Why do we use it?

JavaScript is loose. You can set a variable to a number `x = 5`, then change it to a string `x = "hello"`. This causes crashes in big apps.

TypeScript adds "Types". If you say `x` is a number, it stays a number. It helps catch errors while you write code, not when the user is running it.

Why convert back to JS?

Browsers (like Chrome, Edge) do not understand TypeScript. They only speak JavaScript.

So, we possess a compiler (in the build process) that "transpiles" TS code down to standard JS that the browser can read. This happens automatically when you run ng serve.
